# vite-plugin-debug-meta

Легковесный Vite-плагин, который внедряет отладочные метаданные (пути к файлам и названия компонентов) в React JSX/TSX элементы во время разработки. Это позволяет инспектировать компоненты на странице и открывать их в вашем редакторе кода в один клик.

## Возможности

- 🔍 **Режим инспектирования:** зажмите `Ctrl + Shift`, чтобы войти в режим инспектирования. Наводите курсор на элементы, чтобы увидеть их границы, названия компонентов и размеры.
- 🖱️ **Переход к коду:** используйте `Ctrl + Shift + Клик` на любом элементе, чтобы открыть его исходный файл непосредственно в редакторе кода (VS Code, Cursor, WebStorm, Rider и др.).
- 📋 **Копирование пути к файлу:** автоматически копирует путь к файлу и номер строки (например, `src/components/Button.tsx:25`) в буфер обмена.
- 🌳 **Стек компонентов:** логирует иерархию компонентов в консоль браузера.

## Установка

Установите пакет с помощью npm, yarn или pnpm как dev-зависимость:

```bash
npm install -D vite-plugin-debug-meta
```

или

```bash
yarn add -D vite-plugin-debug-meta
```

или

```bash
pnpm add -D vite-plugin-debug-meta
```

## Использование

Добавьте плагин в ваш `vite.config.ts` или `vite.config.js`:

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { debugMetaPlugin } from "vite-plugin-debug-meta";

export default defineConfig({
  plugins: [
    react(),
    debugMetaPlugin({
      // Необязательно: укажите предпочитаемый редактор (например, 'code', 'cursor', 'webstorm', 'rider')
      // Если не указано, будут использоваться настройки системы по умолчанию или переменные окружения
      editor: "code"
    })
  ]
});
```

### Параметры

| Параметр | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `editor` | `KnownEditor \| string` | `undefined` | Редактор для открытия файлов. Например, `"code"`, `"cursor"`, `"webstorm"`, `"rider"`, `"zed"` и т. д. |

## Как это работает

1. **Babel AST Трансформация:** на этапе `transform` в Vite плагин парсит TSX/JSX файлы и автоматически добавляет атрибуты `data-debug-file` (имя файла и строка) и `data-debug-component` (название компонента) к каждому открывающему JSX-элементу.
2. **Скрипт интерактивного интерфейса:** плагин внедряет глобальный слушатель событий в режиме разработки, который отслеживает зажатие клавиш `Ctrl + Shift`.
3. **Интеграция с редактором:** используется встроенный HTTP-эндпоинт Vite `/__open-in-editor` для запуска редактора с указанием нужного файла и строки.
