# vite-plugin-debug-meta

Легковесный и мощный Vite-плагин для React, внедряющий отладочные метаданные в JSX/TSX элементы на этапе разработки. Он предоставляет интерактивный интерфейс инспектирования элементов на странице, просмотра исходного кода прямо в браузере, отслеживания рендеров React, интеграции с Git и быстрого перехода к исходному коду в вашей IDE в один клик.

---

## 🚀 Основные возможности

### 🔍 1. Режим инспектирования (`Ctrl + Shift` + наведение)
* Зажмите комбинацию клавиш `Ctrl + Shift` для входа в режим инспектирования.
* При наведении курсора элементы подсвечиваются рамкой, а рядом отображается плавающий бейдж с **именем компонента** и его **размерами** в пикселях.

### 🖱️ 2. Быстрый переход в IDE (`Ctrl + Shift` + Клик)
* Кликните левой кнопкой мыши по подсвеченному элементу с зажатыми `Ctrl + Shift`.
* Плагин автоматически скопирует путь к файлу с номером строки в буфер обмена (например, `src/components/Button.tsx:24`), покажет красивое уведомление (toast) и откроет этот файл в вашей IDE на нужной строке.

### 🛠️ 3. Интерактивное меню отладки (`Ctrl` + Правый клик)
Нажмите правой кнопкой мыши на элементе с зажатой клавишей `Ctrl`, чтобы открыть многофункциональное контекстное отладочное меню:

* 🌳 **Стек компонентов с ролями:** Отображает полную иерархию компонентов от корня до выбранного элемента. Компоненты автоматически классифицируются по ролям:
  * `Реализация` (Implementation) — где компонент объявлен.
  * `Использование` (Usage) — где компонент вызывается.
  * `Ветка` (Ancestor) — родительские компоненты.
  * `Смешанный` (Mixed).
  * Каждая роль имеет цветовую кодировку для быстрого сканирования структуры.
* 📝 **Встроенный просмотрщик кода:** Нажмите на иконку `>` рядом с любым компонентом в списке, чтобы развернуть панель с исходным кодом этого файла прямо внутри меню. Целевая строка кода будет автоматически подсвечена.
* 🎨 **Подсветка синтаксиса:** Встроенный быстрый парсер подсвечивает синтаксис исходного кода (ключевые слова, строки, комментарии, теги, атрибуты, функции).
* 🌓 **Переключатель тем (Premium Dark & Light):** Кнопка в шапке меню позволяет мгновенно переключаться между темным интерфейсом (стекломорфизм с размытием заднего плана) и светлой темой с высокой контрастностью.
* 👻 **Эффект прозрачности:** Зажмите клавишу `Ctrl` при открытом меню, чтобы сделать его полупрозрачным и рассмотреть элементы интерфейса, находящиеся под ним.

### 🐙 4. Интеграция с Git (Git Blame & Git Heatmap)
Прямо в панели просмотра кода доступны инструменты анализа Git-истории:
* **Заголовок Git:** Показывает текущую ветку, автора последнего коммита для этого файла и тему коммита.
* **Всплывающий Git Blame:** Наведите курсор на любую строку кода, чтобы увидеть детальную информацию о коммите (автор, хэш, дата коммита и сообщение).
* **Тепловая карта Git (Git Heatmap):** Левая граница строк и их фон подсвечиваются цветом в зависимости от возраста коммита (от теплого оранжевого для недавних изменений до холодного серого/зеленого для старых строк), позволяя мгновенно оценить "свежесть" кодовой базы.

### 🔥 5. Визуализатор рендеров React (Render Visualizer)
* Отслеживает рендеры компонентов React во время взаимодействия с интерфейсом.
* При повторном рендере компоненты кратковременно подсвечиваются вспышкой цвета на странице.
* В отладочном меню отображается счетчик рендеров (`⚡ 3`) и цветовой индикатор частоты рендеров (от *Отлично* до *Критично (Много рендеров!)*).
* Включается и выключается в один клик кнопкой с иконкой пламени в шапке меню.

## 📦 Установка

Установите пакет в качестве dev-зависимости с помощью вашего пакетного менеджера:

```bash
npm install -D vite-plugin-debug-meta
# или
yarn add -D vite-plugin-debug-meta
# или
pnpm add -D vite-plugin-debug-meta
```

## ⚙️ Использование

Добавьте плагин в конфигурационный файл `vite.config.ts` или `vite.config.js`:

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { debugMetaPlugin } from "vite-plugin-debug-meta";

export default defineConfig({
  plugins: [
    react(),
    debugMetaPlugin({
      // Необязательно: укажите предпочитаемый редактор по умолчанию.
      // Если не указано, плагин попытается автоматически определить IDE 
      // или будет использовать переменную окружения LAUNCH_EDITOR.
      editor: "code"
    })
  ]
});
```

### Параметры конфигурации

| Параметр | Тип | По умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| `editor` | `KnownEditor \| string` | `undefined` | Название редактора для открытия файлов. Поддерживаемые значения: `"antigravity"`, `"code"` (VS Code), `"cursor"`, `"trae"`, `"zed"`, `"webstorm"`, `"phpstorm"`, `"idea"`, `"rider"`, `"clion"`, `"pycharm"` и др. |

## 🛠️ Как это работает под капотом

1. **Babel AST Трансформация:** Плагин перехватывает `.tsx`/`.jsx` файлы на этапе `transform` в Vite. Он парсит код в AST (Abstract Syntax Tree), находит открывающие JSX-теги (пропуская фрагменты и сторонние библиотеки) и автоматически внедряет атрибуты `data-debug-file` (относительный путь к файлу и строка) и `data-debug-component` (название компонента).
2. **Служебные эндпоинты Vite Server:** Плагин регистрирует Node.js-посредники (middlewares) в сервере Vite для обработки локальных запросов из браузера:
   * `/__open-in-editor` — для запуска IDE через пакет `launch-editor`.
   * `/__get-source-code` — для чтения исходного кода файла и его передачи в браузерное отладочное меню.
   * `/__get-git-info` — для запуска локальных команд Git (`git blame`, `git log`) и передачи информации о коммитах и ветке в клиентскую часть.
3. **Клиентский интерактивный скрипт:** В HTML-код страницы инжектируется легковесный JS-скрипт и CSS-стили. Скрипт отслеживает клавиатурные события и жесты мыши, строит интерактивное меню, опрашивает React Fiber дерево (для получения точной иерархии компонентов) и взаимодействует с сервером Vite.
